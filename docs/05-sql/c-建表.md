# 表操作

表是数据存储的基本单位。

![](https://raw.githubusercontent.com/hughxusu/lesson-knowledge/develop/images/mysql/component-of-a-database-table.gif)

* 行（Row/Record/Tuple）：表中的一条具体记录，即一条数据。
* 列（Column/Field/Attribute）：表中的一个字段，定义了数据的类型和属性。

查询当前数据库所有表，在终端中可以输入

```sql
\dt
```

## 创建表

使用`create`命令可以创建一张表

```sql
create table users(id int, name varchar(50), gender varchar(1));
```

* `table users`表示创建一个用户表。
* `id`表示字段名；`int`表示字段类型。
* 不同的字段之间，用`,`分开。
* 上面创建的表默认在public schema中。

可以给表和字段添加注释

```sql
comment on table users is '用户';
comment on column users.name is '用户名';
```

PgSQL字段名的基本规则

1. 长度限制：最多63个字符。
2. 合法字符：字母、数字、下划线。
3. 不能以数字开头。
4. 不能是PgSQL保留字。
5. PgSQL中使用其它字符作为标识符，需要用双引号（**不推荐使用**），如：

```sql
"user-name"
"user@email"
"user.age"
```

> [!note]
>
> PgSQL中单引号与双引号
>
> * 单引号用于标识实际的值。
> * 双引号用于标识一个关键字，如：表名、字段名等。

### 查看指定表结构

查看表的字段，字段的类型、是否可以为NULL，是否存在默认值等信息，可以在终端中输入

```sql
\d users;
```

## 数据类型

在前面创建表语句中，用到了`int`和`varchar`来定义数据类型。除此之外，在MySQL中还有多种其它的数据类型。MySQL的数据类型，主要分为三类：数值类型、字符串类型、日期时间类型。

### 数值类型

| 类型             | 大小 (bytes) | 有符号(SIGNED)范围             |
| ---------------- | ------------ | ------------------------------ |
| SMALLINT         | 2            | `(-32768, 32767)`              |
| INT/INTEGER      | 4            | `(-2147483648, 2147483647)`    |
| BIGINT           | 8            | `(-2^63, 2^63-1)`              |
| REAL             | 4            | `(-3.402 E+38, 3.402 E+38)`    |
| DOUBLE PRECISION | 8            | `(-1.798 E+308, 1.798 E+308)`  |
| DECIMAL/NUMERIC  |              | 依赖于M（精度）和D（标度）的值 |

* `DECIMAL`用于存储精确数值的数据类型，特别适合财务、货币等需要精确计算的场景。

```sql
# DECIMAL(M, D)
DECIMAL(5, 2) # 总位数5，其中整数部分最多3位，小数部分2位，有效数字范围：-999.99 到 999.99
DECIMAL(3, 0) # 总位数3，没有小数部分，有效数字范围：-999 到 999
```

* M（Precision-精度）：总位数，范围1~65。
* D（Scale-标度）：小数位数，范围0~30，且D≤M。

### 字符串类型

| 类型    | 大小                    | 描述                        |
| ------- | ----------------------- | --------------------------- |
| CHAR    | 固定`n`字符对应的字节数 | 定长字符串（需要指定长度)） |
| VARCHAR | 实际字符数对应的字节数  | 变长字符串（需要指定长度）  |
| TEXT    | 可以存储1G内容          | 长文本数据                  |

* `char`是定长字符串，指定长度多长，就占用多少个字符，如果长度不够，会用空格补充。
* `varchar`是变长字符串，指定的长度为最大占用长度。
* 实例
  * `username varchar(50)`用户名长度不定，但最长不会超过50个字符。
  * `gender char(1)`性别男或女，只要一个字符。
  * `phone char(11)`手机号，固定一个字符。

> [!note]
>
> 数据类型的选择，在够用的前提下，越小越好，这样可以更多的节省存储空间。

### 日期时间类型

| 类型      | 大 小 | 范围                                       | 格式                  | 描述   |
| --------- | ----- | ------------------------------------------ | --------------------- | ------ |
| DATE      | 3     | 1000-01-01<br>9999-12-31                   | `YYYY-MM-DD `         | 日期   |
| TIME      | 3     | -838:59:59<br>838:59:59                    | `HH:MM:SS`            | 时间   |
| TIMESTAMP | 4     | 1970-01-01 00:00:01<br>2038-01-19 03:14:07 | `YYYY-MM-DD HH:MM:SS` | 时间戳 |

* 实例：`birthday datetime`生日，记录出生年月，可以用来计算年龄。

### 其他数据类型

* `boolean`布尔类型，表示ture和false，占一个字节。
* `BIT(M)`位数据类型，用于存储位二进制值。
* `ENUM`枚举类型，用于存储单一值，可以选择一个预定义的集合。
  * 定义枚举类型：`create type gender_enum as enum('男', '女', '其他', '保密');`
    * 定义`gender_enum`可以和其他的类型一样使用。
    * 值必须用单引号括起来，并用逗号分割开。
    * 实际存储的是值的索引`(1, 2, 3...)`，不是字符串本身。


* `JSON`和`JSONB`数据类型。
  * `JSON`就是json的字符串类型。
  * `JSONB`是json的二进制类型，便于加索引。

[PgSQL数据类型手册](https://www.postgresql.org/docs/current/datatype.html)

> [!warning]
>
> 对于图片、音频、视频等文件，不存储在数据库中，而是上传到某个服务器上，然后在表中存储这个文件的保存路径。

## 约束

约束是作用于表中字段上的规则，用于限制存储在表中的数据。以保证数据库中数据的正确性、有效性和完整性。常见的几种约束

| 关键字        | 名称     | 描述                                           |
| ------------- | -------- | ---------------------------------------------- |
| `NOT NULL`    | 非空约   | 限制该字段的数据不能为空                       |
| `UNIQUE`      | 唯一约束 | 保证该字段的所有数据都是唯一的                 |
| `PRIMARY KEY` | 主键约束 | 主键是一行数据的唯一标识，要求非空且唯一       |
| `DEFAULT`     | 默认约束 | 保存数据时，如果未指定该字段的值，则采用默认值 |
| `CHECK`       | 检查约束 | 保证字段值满足某一个条件                       |
| `FOREIGN KEY` | 外键约束 | 用来让两张表的数据之间建立连接，保证数据的一致 |

## 修改表

使用`alter table`可以实现对表的修改操作

### 添加字段

```sql
alter table users add phone char(11) unique;
```

* `users`表名。
* `add`关键字，表示添加。
* `phone`字段名称。
* `char(11)`数据类型。
* `comment '手机号'`注释。
* `unique`约束条件。

### 修改数据类型

```sql
alter table users alter column id type bigint;
```

* `column`关键字标准操作列。
* `id`字段名称
* `bigint unsigned`修改后的数据类型。

```sql
alter table users add primary key (id);
```

* `add primary key (id)`将`id`列设为主键。

```sql
alter table users alter column id add generated by default as identity;
```

* `generated by`有规则控制生成。
* `default`默认值。
* `identity`数据自动增长。

> [!warning]
>
> 在PgSQL中将已有字段变为主键，需要分为上面3步。

### 修改字段名

```sql
alter table users rename column name to nickname;
```

* `rename`关键字，表示重命名。系统要求`rename`必须是一个独立的原子指令。
* `name`旧列名；`nickname`新列名。

### 删除字段

```sql
alter table users drop column gender;
```

* `drop`关键字，删除。
* `gender`字段名称。

### 修改表名

```sql
alter table users rename to music_users;
```

* `rename to`关键字，表示重命名。
* `music_users`新表名。

### 删除表

```sql
drop table if exists music_users;
```

* `drop table`关键字，删除表。

> [!alert]
>
> 在删除表的时候，表中的全部数据也都会被删除。

## SQL语言的设计哲学

> [!think]
>
> 从上面的语句中可以观察到什么现象？

SQL语言的特点：

1. 动词先行：SQL语句总是以动词开头，这种设计哲学强调了意图的优先级。
2. 动词+对象+条件（限定词）：语法顺序极其接近英语。

SQL的设计哲学被称为“声明式编程”，SQL侧重于告诉计算机需要什么。像Python等编程语言，称为过“过程式”语言，告诉计算机怎么做。

| 概念 | 词性分类   | 核心关键字示例                | 作用                                                         |
| ---- | ---------- | ----------------------------- | ------------------------------------------------------------ |
| 操作 | 动词       | `DELETE`, `CREATE`, `ALTER`等 | 发起意图：决定对系统执行什么指令。                           |
| 对象 | 名词       | `DATABASE`, `TABLE`等         | 操作载体：动词作用的目标。包括：实体、容器、数据元           |
| 限定 | 介词、连词 | `FROM`, `WHERE`等             | 逻辑约束：定义动作发生的“范围”、“条件”和“关联方式”。         |
| 修饰 | 形容词     | `UNIQUE`, `PRIMARY`等         | 属性描述：描述对象或数据的特质，定义数据的约束、唯一性等规则。 |

> [!note]
>
> SQL语言操作数据库的步骤：
>
> 1. 先定动作。
> 2. 再找目标。
> 3. 最后细化范围。
