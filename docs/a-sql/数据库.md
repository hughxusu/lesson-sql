## 基础

### E-R模型

* E表示entry，实体。可以理解为类。
* R表示relationship，关系。多个实体间的数据。
* 一个实体转换为数据库中的一个表。表的列，可以理解为类的属性。
* 关系表示两个实体间的对应规则，关系也可以理解为表中的一个列，包括：
  * 一对一
  * 一对多
  * 多对多
* 关系转换为数据库表中的一个列。关系型数据库中的一行就是一个对象。

### 三范式

* 第一范式：列不可以拆分。
* 第二范式：唯一标识。
* 第三范式：引用主键。
* 后一个范式都是在前一个范式的基础上建立的。

### 逻辑删除

对于重要数据会设置一个`isDelete`的列，类型是`bit`，表示逻辑删除。

### 连接

连接不同表的内容进行查询。

```mysql
# 连接查询
select sutdents.name, subject.title, score.score
from scores
inner join students on scores.stuid=students.id
inner join subjects on scores.subid=subjects.id;

# 同样连接查询，另一种写法
select sutdents.name, subject.title, scores.score
from students
inner join scores on scores.stuid=students.id
inner join subjects on scores.subid=subjects.id; # on 后面的内容应该与subjects有关
```

| 语法         |        | 说明                                                   |
| ------------ | ------ | ------------------------------------------------------ |
| `inner join` | 内连接 | 连接表中的数据全部存在才会出现。                       |
| `left join`  | 左连接 | 以左表数据为准，包括两个表都有的数据和左表独有的数据。 |
| `right join` | 右连接 | 以右表数据为准。                                       |

### 视图

```mysql
create view v_stu_sub_sco as
```

### 事物

* 当一个业务逻辑需要多个sql完成时，其中某一条sql语句出错，则希望整个操作都退回。
* 使用事物可以完成退回功能，保证业务逻辑的正确性。
* 事物的四大特征：
  * 原子性：事物中的全部操作不可分割。
  * 一致性：几个并行执行的事物，执行结果必须月按某一顺序串行执行的结果相一致。
  * 隔离性：事物的执行不受其他事物的干扰。
  * 持久性：对于以提交的事物，系统必须保证该事物对数据库的改变不被丢失。
* 表的类型必须是innodb或bdb，才能使用事物。innodb只会对操作数据库的行加锁。

```mysql
# 修改表的名字
alter table sutdents engine=innodb;

begin # 开启
commit # 提交
rollback # 回滚

begin;
update students set name='小郭' where id=1;
commit;

begin;
update students set name='郭靖' where id=1;
rollback;
```

### 锁

```sql
--添加读锁：可以并发读，但是不能并发写，读锁没释放之前不能进行写操作。

unlock tables;

--添加写锁：只有锁表的用户可以进行读写操作，其他用户不行。

unlock tables;
```

__Connection对象__：connection 相当于用终端去链接数据库

`conn = connect(参数列表)`

* `port`：mysql端口
* `db`：数据库名称
* `user`：用户名
* `passwd`：链接密码
* `charset`：通信编码方式，通常用utf8

方法

* `close()`关闭链接
* `commit()` 提交。Connection对象默认开启事物功能，修改操作需要提交。
* `rollback()`放弃
* `cursor()`Cursor对象，用于执行Sql语句并获得结果

操作步骤：

1. 创建connect对象。
2. 用connect对象获得cursor。
3. 使用cursor执行sql语句。
4. 获得返回结果。

### 数据类型

- Object ID：每个文档都有属性`_id`，保证文档的唯一性，可以主动设置。如果没有设置，MongoDB为每个文档提供了独特的`_id`，类型为Object ID。objectID是一个12字节的十六进制数：
  - 前4个字节为当前时间戳
  - 接下来3个字节的机器ID
  - 接下来的2个字节中MongoDB的服务进程id
  - 最后3个字节是简单的增量值
- String：字符串，最常用，必须是有效的UTF-8
- Boolean：存储一个布尔值，true或false
- Integer：整数可以是32位或64位，这取决于服务器
- Double：存储浮点值
- Arrays：数组或列表，多个值存储到一个键
- Object：用于嵌入式的文档，即一个值为一个文档
- Null：存储Null值
- Timestamp：时间戳
- Date：存储当前日期或时间的UNIX时间格式

### 聚合

聚合是基于数据处理的聚合管道，每个文档通过一个由多个阶段（stage）组成的管道，可以对每个阶段的管道进行分组、过滤等功能。然后经过一系列处理，输出相应的结果。

常用管道命令：

* `$group`：将集合中的文档分组，可用于统计结果。
* `$match`：过滤数据，只输出符合条件的文档。
* `$project`：修改输入文档结构，如重命名、增加、删除字段、创建计算结果。
* `$sort`：将输入文档排序后输出。
* `$limit`：限制聚合管道返回的文档数。
* `$skip`：跳过指定数量的文档，并返回余下的文档。
* `$unwind`：将数组类型的字段进行拆分。

常用计算命令：

* `$sum`：计算总和，`$sum:1`表示以1倍计数。
* `$avg`：计算平均值。
* `$min`：获取最小值。
* `$max`：获取最大值。
* `$push`：在结果文档中插入值到一个数组中。
* `$first`：根据资源文档的排序获取第一个文档数据。
* `$last`：根据资源文档的排序获取最后一个文档数据。

```shell
# $group中有几个键，结果中就有几个键
db.student.aggregate({ 
	$group:{ # $group表示分组操作
		_id:"$gender", # 将_id根据gender分组，$gender表示需要分组的字段，_id表示分组后返回值，本例中包括：男和女两个字段
		count:{$sum: 1}, # 对每一组进行计数统计
		avg_age:{$avg:"$age"} # 对age字段求平均值，保存到avg_age中。$age表示字段，$avg操作方法，avg_age保存字段
	}

		_id: null, # _id表示多整个文档进行操作

	}

	{$project:{gender:"$_id", count:1, avg_age:"$avg_age", _id:0},
```
